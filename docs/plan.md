# План работ по стабилизации и упрощению

## Цели
- Восстановить работоспособность поиска и API на проде (Netlify).
- Снизить риск таймаутов/флапания, упростить конфиги.
- Подготовить почву для персистентного хранилища.

## Этапы и критерии приёмки

### Шаг 1 — Выровнять маршрутизацию API (критический фикс)
- Изменить клиентский baseURL на `VITE_API_URL || '/api'`.
- На сервере (Express) в режиме Netlify использовать базовый путь `/.netlify/functions/api` вместо `/`.
- Перенести `health` под тот же базовый путь.
- AC:
  - На проде запрос `GET /api/search?query=telegram&region=us` возвращает `{ success: true, data: [...] }`.
  - В UI поиск подсказывает приложения (автокомплит).
  - `GET /api/health` → 200 OK JSON.

### Шаг 2 — Уважать выбранный регион в Google Play парсере
- Убрать жёсткий список регионов; итерироваться по выбранному региону (одному).
- Ввести жёсткие лимиты на пагинацию.
- AC: парсинг из одного региона значительно быстрее; таймаутов меньше.

### Шаг 3 — Определиться со стратегией хранения
- Краткосрочно: отображать результат `parse` напрямую, не полагаясь сразу на `GET /reviews` в serverless.
- Долгосрочно: подключить персистентную БД (например, Postgres/SQLite-as-a-service) и адаптировать фильтры/пагинацию.
- AC: после парсинга данные видны в UI стабильно; при перезагрузке — доступны из БД.

### Шаг 4 — Снизить нагрузку и таймауты
- Ограничить параллелизм/страницы (App Store/Google Play), добавить таймауты/ретраи.
- Рассмотреть Background Functions/очередь для тяжёлых задач.
- AC: длительные операции не упираются в лимиты Netlify, UI остаётся отзывчивым.

### Шаг 5 — CORS и безопасность
- Убрать `credentials: true` при `origin: '*'` либо ввести whitelisting через `ALLOWED_ORIGINS`.
- AC: корректные заголовки CORS, отсутствие предупреждений.

### Шаг 6 — Унификация конфигов/типов
- Единый `debounce`, согласовать порты/README, общий пакет `types` для client/server.
- AC: отсутствуют дубли и рассинхроны в конфигурации.

### Шаг 7 — Логирование и DX
- Флаг `DEBUG`, единый формат ошибок/логов, отключение лишних логов в проде.
- AC: чистые логи, достаточная диагностируемость.

## Тест-план
- DevTools: `GET /api/search`, `GET /api/health` на проде.
- UI: ввод в поиск → видны подсказки; выбор приложения → парсинг из одного региона, отзывчивый UI.
- Экспорт: `POST /api/export` возвращает файл с корректными заголовками.
