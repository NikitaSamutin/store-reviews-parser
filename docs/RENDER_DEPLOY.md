## Шаги деплоя

### 1. Подключите GitHub репозиторий к Render

1. Откройте [Render Dashboard](https://dashboard.render.com/)
2. Нажмите **"New +"** → **"Blueprint"**
3. Подключите ваш GitHub репозиторий
4. Render автоматически найдёт `render.yaml` и создаст 2 сервиса:
   - `store-reviews-api` (Backend)
   - `store-reviews-frontend` (Frontend)

### 2. Настройте переменные окружения

После создания сервисов:

#### Для API (`store-reviews-api`):
Переменные уже настроены в `render.yaml`:
- `NODE_ENV=production`
- `PORT=10000`

#### Для Frontend (`store-reviews-frontend`):
1. Перейдите в настройки сервиса `store-reviews-frontend`
2. Найдите раздел **Environment Variables**
3. Добавьте переменную:
   ```
   VITE_API_URL=https://store-reviews-api.onrender.com/api
   ```
   ⚠️ **Важно**: Замените URL на реальный URL вашего API сервиса (его можно скопировать из дашборда API)

### 3. Запустите деплой

1. Render автоматически начнёт сборку обоих сервисов
2. API будет доступен по адресу: `https://store-reviews-api.onrender.com`
3. Frontend будет доступен по адресу: `https://store-reviews-frontend.onrender.com`

### 4. Проверьте работу

После завершения деплоя:

1. **Health check API**:
   ```bash
   curl https://store-reviews-api.onrender.com/api/health
   ```
   Должен вернуть:
   ```json
   {
     "status": "OK",
     "timestamp": "...",
     "version": "1.0.2",
     "runtime": "render"
   }
   ```

2. **Откройте фронтенд** в браузере
3. **Попробуйте парсинг** — теперь без таймаутов!

## Особенности Render Free Tier

### ✅ Преимущества
- **Нет таймаутов** — парсинг может занимать сколько угодно времени
- **Полная поддержка Node.js** — все библиотеки работают
- **750 часов/месяц** — достаточно для 1 сервиса 24/7
- **Автоматический SSL** — HTTPS из коробки

### ⚠️ Ограничения
- **Засыпает через 15 минут** без запросов
- **Просыпается ~30 секунд** — но мы добавили keep-alive пинг!
- **512MB RAM** — может быть мало для Puppeteer при большой нагрузке

### 💡 Keep-alive решение
Мы добавили автоматический пинг каждые 12 минут:
- Сервер **не заснёт**, пока пользователь на сайте
- При первом заходе показывается лоадер "Сервер просыпается..."
- После пробуждения работает мгновенно

## Troubleshooting

### Проблема: API не отвечает
- Проверьте логи в Render Dashboard → API Service → Logs
- Убедитесь, что `PORT=10000` установлен

### Проблема: Frontend не может подключиться к API
- Проверьте переменную `VITE_API_URL` в настройках Frontend
- Убедитесь, что URL API правильный (без слэша в конце)

### Проблема: Парсинг всё равно падает
- Проверьте логи API — возможно, нехватка памяти
- Рассмотрите переход на Starter план ($7/мес) с 512MB → 2GB RAM
